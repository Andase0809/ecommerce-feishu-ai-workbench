# 项目案例说明

## 项目定位

本项目是一个面向电商运营场景的商品数据分析与飞书自动化工作台。它不是单纯的采集脚本，也不是强调大模型调用，而是把商品数据清洗、规则化 AI 分析、飞书多维表格同步和人工审核流程串成一个可运行的业务工具。

当前演示品类使用台灯，是为了让字段更具体：价格带、商品定位、SKU 变体、主图、参数、标签、竞品状态和运营建议都能在一个清晰场景里展示。项目本身可以迁移到其他电商品类。

## 业务问题

电商运营在整理商品和竞品资料时，常见问题包括：

- 商品信息散落在网页、表格和截图里，难以统一查看。
- 商品主图、标题、价格、SKU、标签、参数和链接没有结构化。
- 竞品对比容易变成主观描述，缺少可追溯字段和人工复核入口。
- 团队协作时需要在飞书里查看、筛选、补录和审核，而不是只拿到本地 JSON。

## 解决方案

项目通过 Python CLI 读取样例或清洗后的商品数据，完成字段校验、规则分析和飞书工作台创建。

核心流程：

1. 输入商品或竞品数据。
2. 使用 Pydantic 做字段校验和结构化。
3. 使用规则模板生成标题方向、卖点方向、竞品共性、差异化机会和人工审核清单。
4. 通过飞书 OpenAPI 自动创建 Base、数据表、字段、记录和视图。
5. 在飞书里展示商品图库、SKU 变体矩阵、竞品分析表和运营建议表。

## 工作台设计

飞书侧的展示重点是让运营人员“一眼能看懂数据状态”：

- 商品主图使用附件字段，便于画册视图展示。
- 商品链接和变体链接使用 URL 字段，方便跳转核对来源。
- 价格使用数字字段，便于排序和后续统计。
- 价格带、商品定位、采集状态、竞品分析状态和审核状态使用单选/多选字段。
- 冗长原始字段默认隐藏，只在原始字段视图保留。
- 竞品未准备完成时，只展示待补充状态，不生成虚假结论。

## 技术栈

- Python 3
- 飞书多维表格 OpenAPI / `lark-oapi`
- `pydantic`
- `python-dotenv`
- `beautifulsoup4`
- `Playwright` / `DrissionPage`
- `pytest`

## 结果边界

本项目可以展示：

- 自动创建飞书商品运营工作台。
- 商品主图、标题、价格、标签和 SKU 变体结构化同步。
- 竞品数据进入统一字段模型后生成定性分析。
- 人工审核清单和风险边界被写入工作台。

本项目不声称：

- 被真实商家或公司上线使用。
- 提升 GMV、转化率、销量或排名。
- 违反平台访问控制或访问规则。
- 使用真实不可公开数据训练或分析。

## 后续可扩展方向

- 接入真实大模型 API，用同一数据模型替换当前规则化分析函数。
- 增加更多电商品类的字段模板。
- 增加飞书仪表盘或图表视图。
- 增加数据导入适配器，例如 CSV、Excel、ERP 导出表。
- 增加人工审核状态回写和变更追踪。
