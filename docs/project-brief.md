# 项目简报：电商商品信息生成助手

## 1. 背景

用户是 2027 届工商管理本科生，求职方向偏电商运营、内容运营、商家/渠道运营、AI赋能运营。已有真实素材包括：

- 小红书摄影账号，主要做私域引流与转化。
- 红果短剧账号“蜜汁剧场”剧集/花絮剪辑协作，有作品页、账号页、分工/聊天记录等证据可提供。
- 东方财富杯省级二等奖项。
- Excel、SPSS、视觉工具、基础 SQL/Python、AI辅助材料整理经验。

旧求职项目“JD-简历匹配分析器”已跑通，但用户明确希望后续主打项目不是求职工具，而是与电商相关的小型便捷AI项目。

## 2. 项目目标

做一个可上传 GitHub、可截图展示、可写入简历的电商AI小工具：

> 输入商品基础信息，输出标题候选、核心卖点、详情页短文案、平台口吻文案、商品标签和人工检查清单。

项目应体现：

- 电商商品信息结构化能力。
- 内容表达与平台口吻适配能力。
- AI辅助生成与人工审核意识。
- Prompt与流程沉淀能力。
- 轻量工具开发与README表达能力。

## 3. 推荐 v0 范围

只做最小可展示版本，避免技术膨胀：

- 输入方式：JSON / CSV / 表单，三选一。
- 输出方式：Markdown报告 / CSV / 简单页面，三选一。
- 商品样例：3-5个 demo 商品。
- 生成内容：
  - 商品标题候选 3 条。
  - 核心卖点 3-5 条。
  - 详情页短文案 1 段。
  - 小红书种草口吻 1 段。
  - 抖音短视频口播口吻 1 段。
  - 电商平台简洁口吻 1 段。
  - 商品标签 5-8 个。
  - 人工检查清单。

## 4. 计划模式必须问清的问题

新对话进入计划模式后，优先确认：

1. 技术形态：Python CLI、Streamlit/Gradio 页面、还是纯前端小页面。
2. 是否接入真实大模型 API；如果不接入，则先做 prompt模板 + mock输出。
3. 商品品类：优先做摄影/数码配件、家居小物、服饰美妆，还是通用 demo。
4. 项目展示方式：GitHub README截图、网页截图、命令行输出截图，还是三者都要。
5. 简历项目写法：更偏电商运营、内容运营，还是AI赋能运营。

## 5. 推荐默认方案

如果用户没有强偏好，建议默认：

- 技术形态：Python CLI + Markdown输出，后续再升级轻量网页。
- API策略：v0不强依赖真实API，先用可替换的生成函数和Prompt模板；后续可接入 OpenAI/Claude/本地模型。
- 商品品类：先用3个自造电商 demo 商品，包含数码配件、家居用品、摄影服务周边。
- 输出文件：`outputs/sample_product_report.md`。
- 简历定位：电商内容/商品运营AI辅助工具。

## 6. 验收标准

项目完成前不要写入正式简历。完成标准：

- 有清晰 README。
- 有样例输入。
- 有样例输出。
- 有可运行命令或页面。
- 有开源参考说明。
- 不含真实商家敏感数据。
- 不复制开源项目代码。
- 能用 1 分钟讲清“输入-处理-输出-业务价值”。
