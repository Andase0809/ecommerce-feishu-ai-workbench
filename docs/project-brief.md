# 项目简报：电商商品数据分析与飞书自动化工作台

## 1. 背景

本项目面向电商运营、商品运营、内容运营和 AI 赋能运营岗位展示。最初版本是“电商商品信息生成助手”，用于根据 demo 商品数据生成标题、卖点、平台文案、标签和人工检查清单。

v0.1 已扩展为更完整的数据工作台：将商品数据、SKU 变体、竞品资料和规则化分析结果同步到飞书多维表格，让运营人员可以在飞书中查看、筛选、审核和跟进。

## 2. 项目目标

做一个可上传 GitHub、可截图展示、可写入简历的电商数据化工具：

> 输入商品或竞品数据，完成字段清洗、规则化分析和飞书工作台同步，输出可视化、多视图、可人工审核的运营数据工作台。

项目体现：

- 电商商品信息结构化能力。
- SKU、价格带、商品定位、标签和审核状态建模能力。
- 竞品对比与运营建议的定性分析能力。
- 飞书多维表格 OpenAPI 自动化能力。
- 数据安全、脱敏发布和人工审核边界意识。

## 3. 当前范围

### v0：商品信息生成

- 输入 demo 商品 JSON。
- 输出商品标题、核心卖点、平台口吻文案、标签和人工检查清单。
- 可同步到飞书多维表格。

### v0.1：商品运营视觉工作台

- 输入清洗后的店铺商品数据。
- 自动创建运营总览、店铺主要商品、SKU 款式变体、竞品任务和运营建议表。
- 支持附件主图、URL 字段、价格数字、彩色标签、画册视图和筛选视图。

### v0.1：竞品分析工作台

- 输入 1 个主商品 URL 和 10 个竞品 URL。
- 输出本店商品表、竞品商品表、竞品对比分析表和运营建议表。
- 使用规则化模板生成竞品共性、主商品差异点、机会方向、标题方向、详情页方向和人工审核清单。

## 4. 技术栈

- Python 3
- Pydantic
- pytest
- python-dotenv
- 飞书多维表格 OpenAPI / lark-oapi
- BeautifulSoup
- Playwright / DrissionPage

## 5. 展示方式

推荐 GitHub 和作品集展示：

- README 展示项目定位、技术栈、工作流和运行命令。
- 使用 `samples/*.example.json` 和 `outputs/sample-*.json` 展示脱敏样例。
- 使用飞书多维表格截图展示商品图库、SKU 变体矩阵、竞品工作台和运营建议表。
- 使用终端截图展示一条命令完成 dry-run 或飞书同步。

## 6. 真实性边界

- 不把未完成项目写入正式简历。
- 不虚构真实商家使用、GMV、转化率、销量、排名或节省时长。
- 不提交 `.env`、本地浏览器缓存、真实飞书 token、真实采集结果或未脱敏截图。
- 台灯只是演示品类，项目核心是商品数据清洗、规则化 AI 分析和飞书自动化同步。
- 浏览器自动化只作为人工可控的数据整理辅助，发布材料强调遵守平台访问规则。
