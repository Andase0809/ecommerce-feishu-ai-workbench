# 简历价值与写法边界

## 写入简历前提

只有当项目实际完成并可展示后，才允许写入简历。最低要求：

- GitHub仓库或本地可展示目录。
- README。
- 样例输入。
- 样例输出。
- 运行截图或输出报告。
- 开源参考说明。

## 推荐项目名称

```text
电商商品数据分析与飞书自动化工作台｜个人 AI 运营项目
```

## 技术栈写法

```text
Python 3、Pydantic、pytest、飞书多维表格 OpenAPI / lark-oapi、python-dotenv、BeautifulSoup、Playwright / DrissionPage
```

## 可写入简历的方向

项目完成后，可写作：

```text
电商商品数据分析与飞书自动化工作台｜个人 AI 运营项目

- 围绕商品上新整理、SKU 变体管理和竞品对比场景，设计商品数据结构化流程，将商品名称、价格带、定位、主图、链接、标签、采集状态和审核状态等字段清洗后同步至飞书多维表格。
- 使用 Python CLI 与飞书 OpenAPI 自动创建商品工作台和竞品分析工作台，支持附件主图、URL 字段、彩色标签、筛选视图、图库视图、竞品任务表和运营建议表。
- 基于同价位竞品数据生成竞品共性、主商品差异点、机会方向、标题方向和详情页优化建议，并设置人工审核清单，避免生成无法证明的销量、排名、转化率等结论。
```

如果需要强调 v0 的商品文案生成能力，可以追加：

```text
- 在 v0 阶段实现商品标题、核心卖点、平台口吻文案、标签和人工检查清单生成，后续将同一数据模型扩展为飞书运营工作台。
```

## 面试表达口径

推荐口径：

> 这个项目不是为了展示“我会写爬虫”，而是为了把电商运营里的商品整理、SKU 管理、竞品对比和飞书协作流程拆清楚。我先用规则化分析跑通数据结构、飞书字段和人工审核边界，再保留后续替换为真实大模型 API 的接口空间。台灯只是演示品类，项目核心是商品数据清洗、AI 分析和飞书自动化同步。

## 不要这样说

- 不说项目已经被真实商家使用，除非真的发生。
- 不说提升转化率、节省多少时间，除非有真实记录。
- 不说完全原创算法。
- 不说直接使用了某开源项目，除非确实按许可证使用并注明来源。
- 不把京东公开页面数据说成自己的真实店铺数据；展示时需说明为公开页面采集与脱敏样例。
- 不把某个平台或某个品类写成项目唯一范围；京东台灯只作为演示样例，项目核心是商品数据清洗、AI 分析和飞书多维表格同步。
- 不把平台访问控制处理、访问限制处理或登录态复用写成项目亮点。
