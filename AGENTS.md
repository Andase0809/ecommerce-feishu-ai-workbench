# 开发约定

## 项目定位

本仓库用于维护电商商品数据分析与飞书自动化工作台。核心能力包括商品数据建模、模型服务增强、竞品分析、飞书多维表格同步和本地可验证输出。

## 数据边界

- 不提交 `.env`、浏览器缓存、真实飞书 token、真实采集结果或本地输出文件。
- 真实 URL 文件使用 `samples/*.local.json`。
- 公开样例数据应使用脱敏或构造数据。
- 分析结果不输出销量排名、GMV、转化率等不可验证指标。

## 开发原则

- 保持 CLI 可运行，新增功能需要支持 dry-run。
- 外部服务调用需要具备失败降级，不能破坏本地 JSON 输出。
- 飞书字段、视图和附件上传通过 schema 层描述，不在业务逻辑里拼接 OpenAPI payload。
- 模型服务通过 `AIClient` 接入，避免在业务模块中直接写 HTTP 请求。
- 新增数据字段时同步更新样例输入、样例输出和测试。
