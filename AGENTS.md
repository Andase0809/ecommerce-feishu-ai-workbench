# 电商AI项目工作区规则

## 沟通与工作方式

- 默认使用中文沟通。
- 这是用于落地“电商商品信息生成助手”的项目工作区。
- 新对话启动后，先阅读 `AGENTS.md`、`README.md`、`docs/project-brief.md`、`docs/open-source-references.md`、`docs/resume-value.md`。
- 用户计划在新对话中再次进入计划模式；在计划模式内先问清项目边界、技术路线和展示方式，再实施。

## 项目真实性边界

- 不把未完成项目写入正式简历。
- 项目完成并可展示后，才允许沉淀为简历项目经历。
- 不虚构用户量、转化率、节省时长、上线平台、公司使用场景或真实商品数据。
- 可以使用自造样例商品数据，但必须在 README 中说明为 demo 数据。

## 开源借鉴边界

- 允许参考开源项目的功能拆分、README结构、交互方式、Prompt设计思路和技术选型。
- 不直接复制开源项目代码后改名包装。
- 如果使用任何开源代码片段、模板或依赖示例，必须检查许可证，并在 README 中保留来源与许可证说明。
- 最终项目要体现自己的业务场景改造：商品标题、卖点、详情页文案、平台口吻文案、标签、人工检查清单。

## 建议技术原则

- 先做可展示 v0，不追求复杂技术栈。
- 优先本地可运行、可截图、可上传 GitHub。
- 推荐形态：Python CLI 或轻量 Web 页面，二选一；不要同时开太多方向。
- 必须包含：README、样例输入、样例输出、运行说明、项目截图或输出报告、开源参考说明。
