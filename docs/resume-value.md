# 简历价值与写法边界

## 写入简历前提

只有当项目实际完成并可展示后，才允许写入简历。最低要求：

- GitHub仓库或本地可展示目录。
- README。
- 样例输入。
- 样例输出。
- 运行截图或输出报告。
- 开源参考说明。

## 可写入简历的方向

项目完成后，可写作：

```text
电商商品信息生成助手｜个人AI项目

- 围绕电商商品上架和内容种草场景，设计商品信息生成流程，输入商品名称、品类、目标人群、核心参数和平台风格后，输出商品标题、核心卖点、详情页短文案、平台口吻文案和商品标签。
- 将商品信息拆分为品类、使用场景、功能卖点、目标人群和内容平台等维度，沉淀Prompt模板和人工检查清单，减少AI生成内容与商品事实不一致的问题。
- 项目包含样例商品数据、运行说明、输出示例和开源参考说明，可迁移至电商运营、商品运营、内容运营和AI赋能业务场景。
```

## 面试表达口径

推荐口径：

> 这个项目不是为了证明我技术很强，而是为了证明我能把电商运营里的一个高频小任务拆清楚：商品信息怎么结构化、不同平台文案怎么区分、AI生成内容怎么人工检查。我的思路是先用样例商品数据跑通输入和输出，再把Prompt、字段和检查清单沉淀下来。它可以迁移到商品上架、内容种草、详情页文案和运营提效场景。

## 不要这样说

- 不说项目已经被真实商家使用，除非真的发生。
- 不说提升转化率、节省多少时间，除非有真实记录。
- 不说完全原创算法。
- 不说直接使用了某开源项目，除非确实按许可证使用并注明来源。
